## 1. Windows CMD에서 Linux 명령어 사용 오류

### 상황
`.env` 파일을 복사하려고 Linux/Mac 명령어 `cp`를 사용했더니 오류 발생.

### 오류
```
'cp'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.
```

### 원인
`cp`는 Linux/Mac 명령어고 Windows CMD에서는 동작하지 않음.

### 해결
Windows CMD에서는 `copy` 사용.
```cmd
copy .env.example .env
```

### 참고 - 자주 쓰는 명령어 차이

| Linux/Mac | Windows CMD | 설명 |
|-----------|-------------|------|
| `cp` | `copy` | 파일 복사 |
| `mv` | `move` | 파일 이동 |
| `rm` | `del` | 파일 삭제 |
| `ls` | `dir` | 폴더 목록 |
| `cat` | `type` | 파일 내용 보기 |
| `mkdir -p` | `mkdir` (단계별) | 폴더 생성 |

> PowerShell을 사용하면 `cp`, `ls` 등 Linux 명령어도 사용 가능.

## 2. 사키하네 후야 YouTube 수집 오류

### 상황
`collect_youtube.py` 실행 시 후야만 404 오류 발생.

### 오류
```
googleapiclient.errors.HttpError: 404 - The playlist identified with
the request's playlistId parameter cannot be found.
```

### 원인
후야는 YouTube 채널 handle(`@SakihaneHuya`)이 존재하지 않고,
공식 채널 URL이 `youtube.com/channel/UC0YQnenKBCu5sGb7H61n6HA` 형태 (채널 ID 직접 방식).
handle로 조회하면 채널을 찾지 못해서 플레이리스트가 없다는 오류가 남.

### 해결
`config.py`에서 후야의 `youtube_handle`을 비우고 `youtube_channel_id`에 직접 입력.
```python
{
    "id": "sakihane_huya",
    "youtube_handle": "",                               # handle 없음
    "youtube_channel_id": "UC0YQnenKBCu5sGb7H61n6HA",  # 직접 입력
    "chzzk_id": "36ddb9bb4f17593b60f1b63cec86611d",
}
```
그리고 `collect_youtube.py`에서 `youtube_channel_id`가 있으면 handle 조회를 건너뛰도록 수정.

## 3. 치지직 VOD 수집 404 오류

### 상황
팔로워 수는 정상 수집되는데 VOD 목록이 전원 404 오류.

### 오류
```
404 Client Error: Not Found for url:
https://api.chzzk.naver.com/service/v2/channels/{id}/videos
```

### 원인
치지직 API가 `/service/v2/` 경로를 더 이상 지원하지 않음.

### 해결
`collect_chzzk.py`의 VOD URL을 `v2` → `v1`으로 변경하고 파라미터 수정.
```python
# 변경 전
f"{CHZZK_API}/service/v2/channels/{chzzk_id}/videos"
f"?sortType=RECENT&pagingType=OFFSET&offset={offset}&size={size}"

# 변경 후
f"{CHZZK_API}/service/v1/channels/{chzzk_id}/videos"
f"?videoType=REPLAY&sortType=RECENT&offset={offset}&size={size}"
```

## 4. 치지직 채널 ID URL에 글자 중복 삽입

### 상황
팔로워 수는 정상인데 VOD 수집 URL에 채널 ID가 이상하게 변형됨.

### 오류 URL 예시
```
/channels/45e71a766e949e16a34764deb962f9d9f   ← 33자 (비정상)
/channels/45e71a76e949e16a34764deb962f9d9f    ← 32자 (정상)
```

### 원인
`config.py`의 채널 ID 자체는 정상(32자)이었으나,
VOD API URL(`v2`)이 404를 반환하면서 오류 메시지에 ID가 잘못 출력된 것.
실제로는 **API 경로 문제**(3번 항목)가 원인이었음.

### 해결
3번과 동일하게 `v1` API로 변경하여 해결.

## 12. GitHub Actions git push 충돌 오류

### 상황
Actions를 여러 번 실행했더니 GitHub 서버에 커밋이 쌓여서 push 충돌 발생.

### 오류
```
! [rejected]        main -> main (fetch first)
error: failed to push some refs to 'https://github.com/coding-jhj/Stelive_data'
hint: Updates were rejected because the remote contains work that you do not
hint: have locally. This is usually caused by another repository pushing to
hint: the same ref.
Error: Process completed with exit code 1.
```

### 원인
Remote(GitHub 서버)에 로컬에 없는 커밋이 존재하여 push가 거부됨.
워크플로우가 동시에 여러 개 실행되거나, GitHub Pages 배포 등 다른 작업이
같은 브랜치에 커밋을 만들면서 충돌 발생.

### 해결 시도 1 — `git pull --rebase` 추가 (실패)
```yaml
git pull --rebase origin main
git push
```
결과: 동일한 오류 지속. pull 자체도 충돌로 실패.

### 해결 시도 2 — `git push --force` 로 변경 (진행 중)
```yaml
- name: 변경 커밋 & 푸시
  run: |
    git config user.name "github-actions[bot]"
    git config user.email "github-actions[bot]@users.noreply.github.com"
    git add data/
    git diff --staged --quiet || git commit -m "data: 자동 수집 $(date +'%Y-%m-%d')"
    git push --force
```

근거:
- 이 repo는 GitHub Actions 봇만 push하는 자동화 전용 repo
- `체크아웃` 단계에서 서버 파일을 전부 받아온 뒤 push하므로 기존 파일 손실 없음
- 강제 push로 충돌 우회 가능

### 배운 것
Actions가 실패해도 data/ 커밋은 쌓임. 중복 실행하면 충돌 날 수 있으니 한 번만 실행하고 기다려야 함.

## 13. `collect_all.py` 에서 `ModuleNotFoundError: No module named 'collector'`

### 상황
`collect_all.py`를 루트(`stelive_v3/`)에서 실행했는데 `collector` 패키지를 찾지 못함.

### 오류
```
ModuleNotFoundError: No module named 'collector'
```

### 원인
`collect_youtube.py`, `collect_chzzk.py` 내부에서 `from collector.config import MEMBERS`로
임포트하는데, `collect_all.py`가 서브프로세스로 실행할 때 PYTHONPATH가 설정되지 않아서
`collector` 패키지를 찾지 못함.

### 해결
`collect_all.py`에서 서브프로세스 실행 시 `PYTHONPATH`를 루트 경로로 주입.
```python
def run(script):
    root = os.path.dirname(os.path.abspath(__file__))  # stelive_v3/
    base = os.path.join(root, "collector")
    env = {**os.environ, "PYTHONPATH": root}           # 핵심
    r = subprocess.run([sys.executable, os.path.join(base, script)], env=env)
```

### 실행 방법
```cmd
cd C:\stelive_v3
python collect_all.py
```

## 15. AI(제미나이)가 `config.py`의 채널 ID를 가짜로 교체

### 상황
제미나이에게 도움을 받은 후 YouTube 수집이 전원 실패.
모든 멤버가 "채널 없음 또는 비공개" 오류 발생.

### 오류
```
⚠️  채널 없음 또는 비공개 (channel_id=UCUjDe_YscK9psW_pXWp3u_Q)
```

### 원인
제미나이가 `config.py`를 수정하면서 실제 채널 ID 대신 가짜 ID를 넣음.
예: 아야츠노 유니의 실제 채널 ID와 다른 값이 삽입됨.

### 해결

handle이 없는 사키하네 후야만 직접 ID 입력.
```python

{"youtube_channel_id": "UC0YQnenKBCu5sGb7H61n6HA", "youtube_handle": ""}  # 후야만 직접
```

### 배운 것
AI가 코드를 수정할 때 실제 데이터(채널 ID, API 엔드포인트 등)를 임의로 바꿀 수 있음.
수정 후 반드시 원본 데이터와 비교 확인 필요.

## 16. 치지직 VOD URL에 `//` 이중 슬래시 오류

### 상황
팔로워 수는 정상 수집되는데 VOD만 400 오류. URL에 슬래시가 두 개.

### 오류
```
400 Client Error: Bad Request for url:
https://api.chzzk.naver.com/service/v1/channels//45e71a76e949e16a34764deb962f9d9f/videos
```

### 원인
`CHZZK_API` 전역 변수를 f-string에서 사용할 때, 첫 번째 멤버 이후부터
변수 값이 오염되어 URL이 이중으로 생성되는 현상 발생.
`chzzk_id` 값 자체는 정상(32자, 숨김 문자 없음)이었음.

### 해결
`CHZZK_API` 변수 참조를 없애고 URL을 하드코딩으로 변경.
```python
# 변경 전 (문제)
url = f"{CHZZK_API}/service/v1/channels/{chzzk_id}/videos"

# 변경 후 (해결)
url = f"https://api.chzzk.naver.com/service/v1/channels/{chzzk_id}/videos?videoType=REPLAY&sortType=RECENT&offset={offset}&size={size}"
```

## 17. `googleapiclient` 라이브러리의 `playlistItemss` 오타 버그

### 상황
네네코 마시로, 하나코 나나 YouTube 수집 시 404 오류.
오류 URL을 보면 `playlistItems` 대신 `playlistItemss`로 `s`가 두 개.

### 오류
```
HttpError 404 when requesting
https://youtube.googleapis.com/youtube/v3/playlistItemss?...
```

### 원인
사용 중인 `google-api-python-client` 라이브러리 버전에 내부 버그가 있어서
API 엔드포인트 URL에 `s`가 중복으로 붙는 현상 발생.

### 해결
`googleapiclient` 라이브러리 대신 `requests`로 YouTube REST API를 직접 호출하도록 전환.
```python
# 변경 전 (라이브러리 사용 - 버그 있음)
res = youtube.playlistItems().list(playlistId=playlist_id, ...).execute()

# 변경 후 (직접 REST 호출 - 버그 없음)
res = requests.get(
    "https://www.googleapis.com/youtube/v3/playlistItems",
    params={"playlistId": playlist_id, "key": API_KEY, ...}
)
```
